---
title: Agent → Team Onboarding Flow
description: How a solo agent conversation detects a multi-domain task, gets the user's consent, and bootstraps a multi-agent team — end to end across the agent, the MCP server, and the renderer.
---

A single agent is great for focused work, but some requests span several specialties at once (frontend + backend + database + DevOps). This page traces how ThairaAI turns a normal solo conversation into a **multi-agent team**: the agent recognizes the opportunity, asks the user, and — only on consent — creates the team and navigates the user into it. The flow touches three layers (solo agent, the in-process MCP server, the renderer), so the diagrams below follow one request all the way through.

## Modules involved

| #   | Module                | Location                                           | Responsibility                                          |
| --- | --------------------- | -------------------------------------------------- | ------------------------------------------------------- |
| 1   | **Team Guide Prompt** | `src/process/resources/prompts/teamGuidePrompt.ts` | The team-onboarding rules injected into the solo agent  |
| 2   | **Aion MCP Server**   | `src/process/services/aionMcpServer.ts`            | In-process MCP providing `aion_create_team` + `aion_navigate` |
| 3   | **MCP registration**  | `src/process/agent/acp/mcpSessionConfig.ts`        | Register the Aion MCP into the solo agent session       |
| 4   | **DeepLink extension** | `src/renderer/hooks/system/useDeepLink.ts`        | Handle the `navigate` action, support arbitrary route navigation |

## Boundaries and constraints

- **The Agent never creates a Team automatically** — it must first obtain the user's consent.
- **Route allowlist** — `aion_navigate` only allows navigation to known routes (`/team/:id`, `/conversation/:id`).
- **Summary quality** — the Agent's summary directly determines how well the Lead Agent understands the task; the prompt must guide it to produce a structured summary.
- **Fallback** — when the user declines team mode, the Agent continues to complete the task normally in solo mode.
